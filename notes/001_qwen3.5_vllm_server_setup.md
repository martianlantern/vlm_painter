## Goal
Stand up a vLLM OpenAI-compatible API server for `Qwen/Qwen3.5-0.8B` (a new multimodal VL model with Gated DeltaNet + MoE architecture).

## Key Issues Encountered

### 1. uv inline script dependency resolution failed
- The nightly vLLM wheel on `https://wheels.vllm.ai/nightly` (v0.16.1rc1.dev167) was too old and didn't include `Qwen3_5ForConditionalGeneration` support.
- `transformers` from PyPI (4.57.6) also didn't recognize the `qwen3_5` model type.
- uv couldn't resolve `vllm` + `transformers@git` due to pre-release version conflicts.

### 2. --limit-mm-per-prompt format changed
- Old format: `image=1,video=0` (key=value pairs)
- New format: `{"image": 1, "video": 0}` (JSON)

### 3. --model-impl transformers fallback didn't work
- Even with `--model-impl transformers`, the vLLM transformers backend wrapper couldn't handle `Qwen3_5ForConditionalGeneration`.

## Solution
Installed vLLM and transformers from their respective git main branches system-wide:

```bash
uv pip install --system --break-system-packages "vllm @ git+https://github.com/vllm-project/vllm.git" --prerelease=allow
uv pip install --system --break-system-packages "transformers @ git+https://github.com/huggingface/transformers.git" "qwen-vl-utils>=0.0.14"
uv pip install --system --break-system-packages "fire>=0.7.1"
```

This gave us:
- vllm==0.16.1rc1.dev168+g8b9e8b745 (from git, with Qwen3.5 support)
- transformers==5.3.0.dev0 (from git, with qwen3_5 model type)

The script uses `#!/usr/bin/env -S PYTHONUNBUFFERED=1 python3` instead of `uv run --script` since deps are system-installed.

## Result
- Server runs on `http://0.0.0.0:8000` with OpenAI-compatible API
- Model: `Qwen/Qwen3.5-0.8B` with max_model_len=4096, bfloat16, prefix caching enabled
- GPU: NVIDIA A100 80GB PCIe
- Successfully responds to chat completion requests

## Usage
```bash
./serve_qwen3vl.py  # defaults
./serve_qwen3vl.py --model Qwen/Qwen3.5-0.8B --port 8000 --device 0
```
