# VLM Painter

This is a research project on which I am working on currently for training a vision language model (VLM) like [Qwen/Qwen3-VL-2B-Instruct](https://huggingface.co/Qwen/Qwen3-VL-2B-Instruct) to paint and teach it using RL(e.g GRPO). There are also two papers which are related to this in the `papers` dir which you can refer and take inspiration from. One of the papers of [PaintTransformer](papers/paint_transformer/paint_transformer.md) also has the source code available at [PaintTransformer](PaintTransformer) which we can use for things like stroke images and the rendering patterns, the rasterizer, etc. You can also use any of the skills that are available to you for this project

This is the plan that I have in mind for this project
1. Deciding the prompt format: This phase involves just experimenting with the right image sizes (ideally 256 * 256 should be better), the correct prompt and the structure in which the stroke parameters will the generated by the VLM, which stroke parameters to include (x, y position, rotation, scaling, RGB color) and how to format them so that the VLM can easily generate them and that too with flexibility and diversity, this requires a bit of experimentation and ablations over different prompt structure and manually inspecting the outputs to see that they are diveristy and show some form of correct prompt generation structure. You see RL only works when the model shows some behaviour of that property in the first place, there should be some behaviour of that property like 20% in Pass@k etc. For this you can start a [server](serve_qwen3vl.py) for inference
2. After we have decided on this we can now do a little bit of SFT for the model first so that it always generates with the correct format and see the diversity also of the strokes by rendering the strokes and visually inspecting them. If SFT fails to give good results that means something is wrong either in step 1 or 2 and needs to get that right first
3. This step involves actually training the model with GRPO with a reward that makes the model generate strokes (paintings) that are as close to the original image as possible
